## Context
你們要做的是 Planning Poker 的網頁版，必須可在行動端使用，且同時支援多組團隊平行進行。核心困難在於：
- 投票必須在亮牌前保持隱藏，且亮牌需達到「同時揭示」的感受一致性。
- 每個房間必須完全隔離各自的 Round 狀態、倒數計時、投票結果與計算邏輯。

技術棧假設：
- 前端：`Next.js`（行動端優先的 UI + 與伺服器即時通訊）
- 後端：`Node.js + NestJS`（WebSocket gateway 與房間狀態機）

## Goals / Non-Goals

**Goals:**
- 多個 `Room` 可同時存在，彼此狀態隔離。
- Host 可推進 Round 1/2/3，並在倒數/亮牌階段確保所有參與者顯示一致。
- 支援你們指定的投票卡集合：`0,1,2,3,5,8,13,21,?`。
- 支援你們指定的規則：三連續 Fibonacci 範圍判定、平均計算、失敗後 Round 推進、Round 3 後最高/最低移除收斂、以及 `?` 的無法估算流程。

**Non-Goals:**
- 不預設建立帳號體系與長期歷史保存（初版可用簡單房間碼加入即可）。
- 不預設支援離線投票或跨裝置離線同步。

## Decisions

1. 伺服器端為權威狀態（server-authoritative）
   - 所有投票值由伺服器接收並儲存，但在亮牌前不向房間廣播他人投票內容。
   - 當進入亮牌倒數（3..2..1）時，由伺服器統一發送揭示事件，確保同時揭示的時序來源一致。

2. 房間(Room)與 Round 狀態機由伺服器管理
   - 每個 Room 維護獨立狀態：當前 Round（1/2/3）、投票集合、計時器狀態與結果輸出狀態。
   - 參與者加入後如果已在投票完成但尚未亮牌，依需求只能進入下一輪投票（不參與當前輪的投票）。

3. 通訊方式：WebSocket（含房間事件）
   - 使用 WebSocket 實作即時雙向事件（加入房間、投票送出、狀態推送、倒數與結果廣播）。
   - 在後端以 NestJS 的 WebSocket Gateway 實作「Room 事件」與「揭示/倒數」廣播（避免前端各自計時造成不同步）。

4. 投票與顯示資料流
   - 投票階段：每位使用者只看到「自己的選擇」（或只顯示已完成狀態），不會看到他人投票值。
   - 亮牌後：由伺服器推送每個參與者投了哪張卡 + 彙總結果/提醒。

5. 前後端職責切分
   - 前端（Next.js）負責渲染狀態（投票階段/倒數階段/結果階段）與觸發 Host 的操作（開始下一輪）。
   - 後端（NestJS）為狀態權威來源（server-authoritative），負責計算成功/失敗、處理 Round 1/2/3 與 Round 3 收斂規則，以及控制亮牌時間點。

6. 待估項目／輪次：早停、僅成功平均算「得分」、Host 決定下一次投分（2026-03-26 定案）
   - **「得分」定義**：僅 **三連續 Fibonacci 成功判定後之平均（成功平均）** 算得分；**`?`（無法估算）、失敗提醒、Round 3 收斂後之「無法估算」等，皆不算得分**。
   - **早停**：同一待估項目中，**任一 Round 在亮牌後若判定為成功平均，該項目立即完結**，不再進入後續 Round。
   - **往後輪次**：**僅在該項目尚未取得成功平均時**，Host 才可手動開始下一 Round（Round 2、Round 3）。因此 **Round 3 收斂規則僅在未早停、且實際進入並完成 Round 3 投票與揭示時適用**。
   - **三輪皆無成功平均**：若 Round 1～3 結束後仍從未取得成功平均（可能含多輪 `?`、失敗、Round 3 收斂無結果等組合），**該項目亦視為完結**，進入由 Host 決定之待命狀態。
   - **下一次投分**：項目完結後（無論因早停或三輪走完），**僅能由 Host 明確操作以開始下一次投分**；不需項目名稱／編號，僅重置該房間內與「當前待估項目」相關之狀態（輪次、票、計時、結果等）即可。

## Risks / Trade-offs
- [Timer 精度與多裝置同步] → 倒數由伺服器生成時間基準，前端以伺服器事件時間渲染。
- [晚加入/重連造成狀態混亂] → 伺服器在加入時計算使用者可投的 Round；若已過亮牌階段，使用者進入下一個 Round 的投票等待狀態。
- [房間碼被猜測] → 房間碼需足夠隨機不可預測；避免在未授權時傳回 room 內敏感資訊。

## Migration Plan
不適用（新功能、新增系統）。

- `?` 與失敗時之 UI 文案可再微調（行為以 spec：不構成得分、可續下一輪至 Round 3 為準）。
